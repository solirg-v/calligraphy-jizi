## 🔴 明天开工：Android Canvas 字体彻底修复

**入口**：让 Claude 读 memory `android-canvas-font-fix-plan.md`，里面有完整方案 + 节奏 + 风险点。

**问题概要**：
- Android 真机：首页 CSS 是真荆霄鹏 ✅，但 lightbox 单字 / 导出图片是系统字 ❌
- 根因：`wx.loadFontFace` 在 Android 上只进 WebView 字体栈，不进 Native Skia 栈，所以 Canvas 拿不到
- iOS 上两栈连通，所以一直没事

**v1.1.1 的 `_detectFont` CSS 探测兜底是错的**——前提假设"CSS 能渲染 ⟹ Canvas 也能"在 Android 不成立。要删掉重做。

**修复方案（半天）**：
1. 集字 lightbox 改 CSS `<view>` 渲染（30 分钟）
2. 字组 lightbox 改 CSS（20 分钟）
3. 导出按平台分流：iOS 维持本地 Canvas、Android 走 Vercel API + `@napi-rs/canvas`（3 小时）
4. 清理 `_detectFont` 等无用代码（15 分钟）

**iOS 全程零功能、零体验损失**——这是底线。
